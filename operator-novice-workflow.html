阶段 1

问题界定与前置确认

环节 1、4｜要不要做 + 环境能不能做

依据：Ascend「算子分析」；PyTorch 自定义算子前置条件检查。

1

判断是否需要自研算子

产出：自研 / 等框架 / 组合算子
用户先确认：这个算子框架里有没有？没有的话要不要自己写？

📊 首对「查官方文档」已完成 10 项指标跑分 → 页面四

CUDA 路径
  • 查什么：PyTorch 是否已有原生算子；cuDNN/Triton 是否覆盖
  • 去哪查：
    • 1 · 查官方文档：PyTorch Docs、cuDNN/cuBLAS 列表、Custom Op 教程 目的：拿官方权威信息核实有没有现成算子。 效果：得到可引用的支持结论，决定能否直接停在本环节（不走自研）。 问 AI / 让 AI 做：
      • 问帮我在 PyTorch 官方文档里查 vector add 有没有原生算子，并给出对应文档链接。
      • 问请对照 cuDNN/cuBLAS 算子列表，判断我这个算子有没有高效内置实现。
      • 做总结 Custom Op 教程要点：哪些情况才必须写 CUDA kernel？输出一张决策清单。
    • 2 · 问 AI 助手：是否必须自研、cuDNN 覆盖、能否组合内置算子 目的：用自然语言快速理清「自研 / 组合 / 等框架」三条路。 效果：降低读文档门槛，拿到可执行的排查顺序与决策建议。 问 AI / 让 AI 做：
      • 问我是应用开发者，什么情况下必须自研 CUDA 算子？请给排查顺序。
      • 问cuDNN 能覆盖时还有必要手写 kernel 吗？帮我对比利弊并给建议。
      • 做能否用几个 torch 内置算子组合实现？请给等价写法和判断依据。
    • 3 · IDE 内 Agent：选中代码问 cuDNN 路径、扫仓库、写对比脚本 目的：结合真实工程判断瓶颈，用运行证据验证猜测。 效果：确认慢的是否必须自研，或换内置/组合算子即可解决。 问 AI / 让 AI 做：
      • 问分析我选中的 forward，慢在哪个 op？有没有走 cuDNN？
      • 做扫描当前仓库，列出可替换为 PyTorch 内置的自定义算子。
      • 做生成对比脚本：torch.add vs 我现在的实现，比对输出精度与耗时。
CANN 路径
  • 查什么：torch_npu 是否已支持；CANN 内置算子是否覆盖
  • 去哪查：
    • 1 · 查官方文档：torch_npu 支持清单、CANN 算子库、兼容性列表 目的：核实目标算子在当前 CANN / torch_npu 版本下是否已支持。 效果：确认版本匹配与支持范围，避免在已支持场景误启动 Ascend C 自研。 问 AI / 让 AI 做：
      • 问帮我在 torch_npu 支持清单里查目标算子是否支持，并注明版本要求。
      • 问根据我的 CANN + torch_npu 版本，查兼容性列表并判断能否直接跑。
      • 做在 CANN 内置算子库文档里找等价 vector add 实现，整理对照表。
    • 2 · 问 AI 助手：Ascend 是否支持、替代方案、迁移差异 目的：快速弄清 NPU 侧的替代路径与 CUDA→Ascend 差异。 效果：在「等框架 / 组合算子 / 自研」之间做初步决策，减少盲查文档时间。 问 AI / 让 AI 做：
      • 问如何查某 PyTorch 算子在 Ascend 上是否已支持？请给逐步排查流程。
      • 问不支持时，等框架 / 组合算子 / 自研 Ascend C 各适合什么场景？帮我选型。
      • 做从 CUDA 迁到 Ascend，这个算子常见坑有哪些？输出检查清单。
    • 3 · IDE 内 Agent：查 NPU fallback、列 torch_npu 调用、NPU 探测脚本 目的：在真实工程与 NPU 运行环境中验证支持情况。 效果：用试跑结果和报错定位不支持的算子，判断必须自研还是改写法即可。 问 AI / 让 AI 做：
      • 问分析选中的 NPU 代码：是否走加速还是 fallback 到 CPU？标出可疑算子。
      • 做扫描项目，列出所有 torch_npu 调用及可能不支持的算子。
      • 做生成 NPU 探测脚本并试跑，整理支持情况与报错信息。

产出：环境 OK / 缺什么组件

写 CUDA 自定义算子需要装哪些？nvcc 和 PyTorch CUDA 版本怎么对应？
CANN 路径
  • 查什么：CANN Toolkit、Ascend C 编译器、torch_npu、soc_version
  • 验证：source set_env.sh；npu-smi info；torch.npu.is_available()
写 Ascend C 算子需要装 CANN 哪些包？soc_version 怎么查？

  • 学什么：thread、block、grid、warp；threadIdx / blockIdx

  • 学什么：Vector 核、Cube 核；UB/GM 内存层级；SPMD 模型
  • 为何学：Ascend C 编程模型与 CUDA 不同，不能照搬
