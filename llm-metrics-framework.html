大模型回答质量 · 度量指标框架

研究背景：开发者在同场景、同任务、同目标下完成算子开发 workflow，对比大模型在 CUDA 与 CANN 两条路径上的回答质量差异与执行效率差异。

页面一：旅程地图 页面二：算子 Workflow 页面三：度量指标 页面四：度量结果

研究设计 · 配对对照

  • 固定变量：场景（如环节 1「判断要不要自研」）、任务（Hello World vector add）、用户角色（新手）、提问意图（同一条「问 / 做」）。
  • 对照变量：技术栈路径 — CUDA（PyTorch + cuDNN/cuBLAS） vs CANN（torch_npu + Ascend C 生态）。
  • 两个因变量：① 质量（内容对不对、全不全、能不能用）；② 效率（多少步、多少轮、多久到结论）。
  • 聚焦维度（主报告高亮）：检索深度与精准度、路径效率、多轮收敛、官方信息占比、信息时效性、知识覆盖率、快速给答（输出层）、代码简洁性、可执行性与完整性。
  • 记录单元：每一对配对问题 → 两次会话（CUDA / CANN 各一）→ 聚焦维度 Δ；非高亮列视任务选填。

执行层 · 效率差异

30%

回答「哪边更快帮用户到达环节产出」。CANN 文档分散、概念断层多，预期在路径效率、多轮收敛上差异最大；检索深度用于解释效率差距来源。

内容层 · 质量差异

45%

本研究核心层。同场景下 CUDA 回答常更 grounded；CANN 侧易出 API 幻觉、漏版本配套、漏 torch_npu 清单。正确性、幻觉、覆盖深度权重最高。

输出层 · 可用性差异

25%

回答「哪边输出更能直接落地」。代码简洁性与快速给答在含代码/决策环节权重提升。

核心指标速查（Gold Set 说明）

与三层表格高亮列主指标一致，建议每条配对必报 Δ。指标名称 hover 可查看补充说明。

全自动 规则+日志/script 可复算   半自动 需预定义 Gold/checklist，部分步骤可脚本   需标注 依赖人工判定或专家 rubric

指标名称 指标定义 计算方法 自动化程度 所属模块 所属层 文献依据
Recall@k 前 k 次检索/引用中，命中环节 Gold 文档集的比例；衡量是否找全官方依据。 Recall@k = |Relevant ∩ Top-k| / |Relevant| 半自动检索 log 可自动解析；Gold 文档集与相关性需预定义 检索深度与精准度 执行层 Manning et al., 2008 经典 IR；Pirolli & Card, 1999 信息觅食
MRR 首个相关 Gold 文档排名的倒数均值；衡量检索排序是否把官方文档排前面。 MRR = mean(1 / rankfirst relevant) 半自动同 Recall@k，依赖 Gold 相关性判定 检索深度与精准度 执行层 Manning et al., 2008 Mean Reciprocal Rank 标准定义
NPL 从提问到环节产出的有效步骤数相对专家最短路径的归一化长度。 NPL = STS / STSbaseline 半自动步数可计 log；「有效步骤」与 baseline 需专家路径 路径效率 执行层 Card et al., 1980 GOMS/KLM 最小操作路径；HCI 任务效率测度
TTC 从提问到出现可停止结论的时间（秒）或对话轮次。 TTC = tconclusion − tquery（或轮次数） 半自动时间戳/轮次客观；「结论出现」需 slot 规则或标注 多轮收敛 执行层 Young et al., 2013 任务型对话；Mehri et al., 2019 USR；Miller, 1968 响应时间
OSR 引用中来自该栈官方一手源的比例（非 CUDA 博客充当 CANN 依据）。 OSR = #official_cites / #total_cites 全自动栈级域名白名单 + URL 抽取即可复算 官方信息占比 内容层 CRAAP 信源评估框架；Epistemic trust；引用权威分析（bibliometrics）
HalRate 编造或不可验证的 API、支持清单、版本信息等占全部可核查陈述的比例。 HalRate = #hallucinated_claims / #total_claims 需标注claim 分解与 API/版本真伪需对照 Gold 或人工核查 信息时效性与 API 幻觉 内容层 Ji et al., 2023 NLG 幻觉综述；HaluEval 实体级 P/R 评估范式
Cov@checklist 回答覆盖环节栈特有检查项（如 torch_npu 清单、CANN 配套）的比例。 Cov@checklist = |hit ∩ required| / |required| 半自动checklist 固定后可关键词/NER；深度分级（DOK）常需标注 知识覆盖率 内容层 Es et al., 2023 RAGAS Context Recall；集合覆盖率；Webb, 2002 DOK 深度分级
pass@1 回答中代码块复制到 sandbox 一次试跑是否成功；衡量能否少改即跑。 pass@1 = #run_ok / #code_blocks 全自动sandbox 试跑，exit code 客观 可执行性与完整性 输出层 Chen et al., 2021 HumanEval pass@k；JTBD 任务完成度
TTFD 单次回答中，决策句（自研/组合/等）首次出现的位置占全文比例。 TTFD = tokensdecision / tokenstotal 半自动token 计数客观；决策句可用规则或双盲标注 快速给答 输出层 Miller, 1968 人机对话响应时间；Grice 合作原则·量准则
CSC 同功能下，回答代码相对参考最简实现是否足够短、足够简单。 CSC = LoCref / LoCanswer 全自动参考实现固定后 LoC 可脚本统计 代码简洁性 输出层 McCabe, 1976 圈复杂度；Clean Code 简洁原则；ISO/IEC 25010 可维护性

Gold Set 由研究者按环节预定义：如环节 1 CUDA 含 {torch.add 文档, cuBLAS L1, Custom Op 教程}；CANN 含 {torch_npu 清单, CANN 配套表, Ascend C 流程}。配对任务共用同构 checklist，栈特有项分开列。完整书目见页底主要理论参考文献。

执行层 · 30%

Process · 效率差异：大模型「找得多快、收敛多快」

主因变量：路径步数、轮次、TTC｜CANN 预期劣势项
检索深度与精准度重点高层内 20% 工具调用质量中层内 25% 路径效率重点高层内 30% 多轮收敛重点高层内 25%
定义 检索是否覆盖该栈决策所需官方源（CUDA：PyTorch+NVIDIA；CANN：torch_npu+昇腾），且页面与算子相关。 Agent 工具调用是否匹配场景（查文档 / 跑命令 / 扫仓库），且栈相关命令可成功执行。 从提问到环节产出（自研/组合/等框架）的有效步骤数；冗余检索与无效生成计入惩罚。 配对任务下，是否在有限轮次内给出可停止的明确结论；多快出现决策句（TTFD）与首响（TTFR）。
核心
指标
  • Recall@k — Gold 文档命中率 必报
  • MRR — 首个相关文档排名 必报
  • nDCG@k — 排序质量
  • Context Precision（RAGAS）
Recall@k = |Relevant ∩ Top-k| / |Relevant|
MRR = 1/rankfirst relevant
  • Tool Success Rate (TSR)
  • Tool Selection Accuracy
  • Invocation Precision / Recall
  • Observation Utilization Rate
TSR = #success / #invocations
Util = cited_obs / total_obs
  • STS Steps-to-Success
  • NPL Normalized Path Length 必报
  • DE Dialogue Efficiency
  • Redundant Step Rate
NPL = STS / STSbaseline
  • TTC Time-to-Conclusion 必报
  • DE Dialogue Efficiency
  • TTFD Time-to-First-Decision
  • TTFR Time-to-First-Response
TTC = tconclusion − tquery；单轮 TTFD 见输出层
对照
Δ 指标		 ΔRecall@k 必报；ΔMRR 必报；ΔnDCG@k 可选。 ΔTSR、ΔUtil：NPU 环境缺失导致的工具失败差。 ΔNPL 必报；ΔSTS、ΔDE 可选。 ΔTTC 必报；ΔDE 可选。
度量
协议		 预构建环节 Gold 文档集（按栈）；标注检索/引用 top-k 相关性（0/1/2）；算 IR 指标。 解析 Agent trace：tool name、args、exit_code；对照 ToolBench 分类（search/code/read）。 从 session log 计有效 action；baseline 取专家最短路径 STSref。 定义环节 slot（如 decision, evidence, version）；标注每轮 slot 填充；Success=产出+正确。
阈值
参考		 Recall@5≥0.8 且 MRR≥0.5 为优；CANN 侧 Recall 常低于 CUDA。 TSR≥0.9；Util≥0.7 表示工具结果真正进入回答。 NPL≤1.2 为优；>2.0 视为路径冗余。 DE≥0.5（≤2 轮成功）；GDR=0 为优。
理论
依据		 经典 IR（Manning et al., 2008）：Recall@k、MRR、nDCG；信息觅食（Pirolli & Card, 1999）：patch 选择效率。 ReAct（Yao et al., 2022）；ToolBench / API-Bank：tool selection & success；Plan-and-Solve 工具链评估。 GOMS / KLM（Card et al., 1980）：最小操作路径；HCI 任务效率：TTC、TTFA 标准测度。 任务型对话（Young et al., 2013；Mehri et al., 2019 USR）：success rate、slot filling；Grice 合作原则： relevance & quantity。
内容层 · 45%

Content · 质量差异：大模型「说得对不对、全不全」

主因变量：事实准确率、栈覆盖｜本研究最核心层
官方信息占比重点高层内 20% 信息时效性与 API 幻觉重点高层内 25% 知识覆盖率重点高层内 30% 正确性与事实性高层内 25%
定义 依据是否来自该栈官方一手源（非把 CUDA 博客当 CANN 依据）。 API / 支持清单 / CANN 版本是否与真实环境一致；编造 torch_npu 支持属致命项。 是否覆盖该环节栈特有检查项（CANN：torch_npu 清单、CANN 配套、NPU 试跑；CUDA：cuDNN 路径）。 可核查陈述的准确率；错误是否导致错误决策（误判已支持→误不自研）。
核心
指标
  • OSR Official Source Ratio 必报
  • Stack-Valid Rate
  • Citation Precision
OSR = #official / #total cites
  • HalRate 幻觉率 必报
  • API Entity P/R/F1
  • VerAcc Version Accuracy
  • Temporal Validity Rate
HalRate = #hallucinated / #claims
P/R vs API Gold Inventory
  • Cov@checklist 必报
  • DWC DOK 加权深度
  • Context Recall（RAGAS）
  • Depth Level L1/L2/L3 分布
DWC = Σ w·DOK / Σ w
L3=给验证步骤, L2=解释, L1=提及
  • Faithfulness（RAGAS）
  • Label Accuracy（FEVER）
  • Claim-level F1
  • Decision Error Rate (DER)
Faith = supported / total_claims
DER = wrong_decision / total
对照
Δ 指标		 ΔOSR 必报；ΔStack-Valid 可选。 ΔHalRate 必报；ΔAPI-F1、ΔVerAcc 可选。 ΔCov@checklist 必报；ΔDWC 可选。 ΔFaithfulness、ΔDER：致命决策错误栈分布。
度量
协议		 维护栈级域名白名单（CUDA/CANN）；抽取 URL 与「据官方文档」声明；CRAAP 分级。 NER 抽取 API/算子名；对照 Gold Inventory + docs 验证；版本对照 release notes。 环节 checklist 二元/三级标注；RAGAS context recall 用 Gold facts 作 reference。 Atomic claim decomposition → SUPPORTS/REFUTES/NEI（FEVER）；DER 仅计影响环节产出的错误。
阈值
参考		 OSR≥0.7 且 Stack-Valid≥0.9；CANN 错引 CUDA 源 → Stack-Valid 下降。 HalRate≤0.05 为优；API-F1≥0.9；VerAcc=1.0 为硬性要求（版本类）。 Cov@checklist≥0.85；Context Recall≥0.8（RAGAS 常用区间）。 Faithfulness≥0.9；DER=0 为优；任一 fatal claim → 样本 invalid。
理论
依据		 CRAAP / 信源评估；Epistemic trust；引用分析（bibliometrics）中 authority 概念。 HaluEval / Hallucination Survey（Ji et al., 2023）；实体级 P/R（信息抽取评估）；软件版本兼容性理论。 RAGAS Context Recall & Answer Relevance（Es et al., 2023）；集合覆盖率；Webb DOK 深度加权。 FEVER（Thorne et al., 2018）；RAGAS Faithfulness；ISO/IEC 25010 功能正确性。
输出层 · 25%

Output · 可用性差异：大模型「用户能不能直接用」

主因变量：可运行率、栈合规｜用户感知补充
认知友好度中层内 15% 解释粒度匹配中层内 15% 代码风格合规中层内 15% 代码简洁性重点高*层内 20% 可执行性与完整性重点高层内 30% 快速给答重点高层内 15%
定义 新手在该栈语境下理解成本；CANN 额外概念（驱动/CANN/torch_npu 三角）是否被解释。 解释深度是否匹配页面二 persona；同问题下 CUDA/CANN 回答结构是否对称（先结论后步骤）。 代码是否用对栈 API（.cuda() vs .npu()、import torch_npu），非复制粘贴 CUDA 代码。 同功能下代码是否足够短、足够简单：LoC、圈复杂度、无冗余 wrapper。 脚本/命令能否少改即跑；是否闭环到环节产出；CANN 是否缺 build/验证步。 多快给出可停止的决策结论（TTFD）；首响（TTFR）与全文冗余（RedRate，信息等价时）。

对照
Δ 指标		 ΔTLX、ΔJD：CANN 概念断层是否提高负荷。 ΔSYM、ΔPGS：配对结构是否同构可比。 ΔSLI、ΔDCR ΔCSC 必报*；ΔCC 可选。 Δpass@1 必报*；ΔWCR、ΔSCR 可选。 ΔTTFD 必报；ΔTTFR 可选。
度量
协议		 目标用户（新手）完成 NASA-TLX；标注 CANN-only 术语是否解释。 预写参考大纲（按 persona）；双盲评 SYM；PGS 用 Likert rubric。 代码块 AST/regex；栈白名单 token；ruff/clang-tidy 可选。 McCabe CC；与 LoCref（专家最简实现）比 CSC。 sandbox 试跑；SCR 对 workflow 模板。 标注 decision 句首位置；记录 TTFR（API 延迟或首段输出时间）。
阈值
参考		 TLX≤50（100 量表）为可接受；JD≤0.05。 SYM≥0.8 方可做公平 Δ 比较；PGS≥4/5。 SLI≤0.05；DCR=1.0 为优（CANN 侧 .cuda() 即违规）。 CSC≥1.0 且 CC≤10 为优；冗长但不简洁 ↓ 分。 pass@1≥0.8；WCR=1.0。 TTFD≤0.25；TTFR 按场景 baseline 比。
理论
依据		 认知负荷理论（Sweller, 1988）；NASA-TLX（Hart & Staveland, 1988）工作负荷标准量表。 最近发展区（Vygotsky）；文本相似度（ROUGE/BLEU, Lin 2004）用于结构对齐；Grice 方式准则。 Norman 概念模型；软件惯例一致性；API 设计 usability（Nielsen heuristics）。 McCabe（1976）圈复杂度 CC；Clean Code；ISO 25010 可维护性。 HumanEval pass@k（Chen et al., 2021）；JTBD 任务完成。 HCI 响应时间（Miller, 1968）；Grice 量准则；对话 latency 评估。
